# Princípio MC #44 v1.0 PROPOSTO — Marca MC vive em dois estados gráficos · Filled primário · Linha derivada canônica

## §1 · Enunciado

A marca Meu Cumpadre (símbolo Arcos+Esferas + wordmark) existe canonicamente em **dois estados gráficos coexistentes**, ambos representando a mesma identidade institucional:

1. **Versão Preta (filled / solid)** · PRIMÁRIA · canônica default
2. **Versão Linha (outline / stroke)** · DERIVADA canônica · uso contextual específico

**Não é hierarquia de importância · é hierarquia de aplicação default.** Quando não houver razão explícita para usar Linha, usar Preta.

## §2 · Fundamentação técnica da escolha "Filled como primário"

### 2.1 · Escalabilidade

| Tamanho | Versão Preta filled | Versão Linha outline |
|---|---|---|
| 16×16px (favicon) | ✅ Lê | ❌ Stroke < 0.5px · desaparece |
| 32×32px (favicon HDPI · avatar WhatsApp redux) | ✅ Lê com clareza | ⚠️ Stroke fica < 1px · perde definição |
| 40×40px (avatar WhatsApp default Zilda) | ✅ Lê com presença | ⚠️ Lê frágil · não comunica força |
| 128×128px (header médio) | ✅ Lê + tem peso | ✅ Lê com elegância |
| 256×256px+ (capa pitch deck · cabeçalho) | ✅ Presença institucional | ✅ Refinamento editorial |
| Outdoor 6×3m | ✅ Funciona | ✅ Funciona |

**Limite técnico Linha:** stroke deve ter ≥ 2pt no master vetor · em escala < 128×128px o stroke fica abaixo do limiar de renderização confiável em telas LCD/AMOLED comuns.

### 2.2 · Audiência Zilda first

No celular antigo da Dona Zilda, com tela arranhada e brilho da luz do sertão, **forma cheia sólida lê antes** que linha fina. Princípio Zilda-STF (D.8.1) prioriza a leitura da Zilda · STF lê qualquer coisa.

### 2.3 · Aplicação Visual Law CP-MC

POP VisualLaw PARTE 2 v1.3 define que **headers de página devem ter peso institucional** (Diamante 700 sobre Papel ou `#D4AF37` sobre Papel). Uma linha fina de outline competiria com texto jurídico denso · uma forma filled em `#D4AF37` ou Diamante 700 ancora o cabeçalho sem competir.

### 2.4 · Histórico institucional de marcas

Marcas que sobreviveram décadas consolidaram filled como primário · linha como derivada para contextos específicos: Mercedes-Benz, Apple, Nike, Twitter/X, IBM, BMW, Audi, NASA. Reproduzir esse padrão é alinhar com o que **funciona** em horizonte longo · evita ruído visual.

## §4 · Regras invioláveis de uso

### 4.1 · Versão Preta NUNCA em outros estilos

- ❌ Versão Preta com gradiente · solid fill apenas
- ❌ Versão Preta com sombra · zero sombra (filled é a sombra simbólica · contradição)
- ❌ Versão Preta com outline adicional (não é "outline+fill" · é fill puro)
- ❌ Versão Preta em cor fora de `#D4AF37` · `#1A120B` · Diamante 700 · branco em dark mode

### 4.2 · Versão Linha NUNCA em outros estilos

- ❌ Linha em escala < 128×128px (stroke morre)
- ❌ Linha com fill semi-transparente (é Linha · não filled)
- ❌ Linha com stroke variável (todo o contorno em stroke uniforme · peso 2pt vetor master)
- ❌ Linha tracejada ou pontilhada (é Linha contínua sólida)
- ❌ Linha com glow ou efeito neon

### 4.3 · Versão Preta E Versão Linha NUNCA simultâneas

A mesma aplicação **não combina** Versão Preta e Versão Linha lado a lado · gera confusão hierárquica. Exceção única: brand book / Estudo 1 do Sistema de Projeto MC · onde **o próprio propósito** é mostrar as duas lado a lado para documentar o sistema.

## §6 · Anti-padrões observados (proibições)

Durante o sweep 18/05 que cunhou este princípio · 3 anti-padrões emergem da reflexão pós-Imagem 3:

### Anti-padrão 1 · Tratar Versão Linha como "marca simplificada para favicon"

❌ ERRADO: "vamos usar a versão linha porque é mais simples para o favicon"
✅ CORRETO: "para favicon, usamos Versão Preta filled porque é a única que renderiza em 16-32px"

A simplicidade da Linha é editorial · não técnica. Linha **NÃO É** "filled de baixa fidelidade" · é estado canônico próprio.

### Anti-padrão 2 · Misturar versões em mesma aplicação

❌ ERRADO: header de página com símbolo Versão Preta + marker Proof-First Versão Linha no mesmo design system de cor
✅ CORRETO: header Versão Preta em `#D4AF37` + marker Proof-First Versão Linha em Diamante 700 · cores **diferentes** assegurando que não competem por hierarquia visual

### Anti-padrão 3 · Inventar terceiro estado

❌ ERRADO: "vamos criar uma versão semi-filled, ou uma versão com gradient, ou uma versão 3D"
✅ CORRETO: 2 estados · 2 estados apenas. Filled ou Linha. Qualquer terceiro estado é desvio.
