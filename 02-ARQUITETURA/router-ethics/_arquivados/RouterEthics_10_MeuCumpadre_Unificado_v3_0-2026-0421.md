# **Capítulo 0 — Manifesto: Por Que o Router-Ethics Existe**

## **0.1 O Asilo Digital Brasileiro**

A digitalização compulsória de serviços estatais — centralizada no portal gov.br e no aplicativo Meu INSS — converteu-se em mecanismo de exclusão social. O sistema exige do hipervulnerável um nível de letramento digital que ele estruturalmente não possui, transformando facilitação em filtro de negação de direitos. Conta gov.br Prata e Ouro, reconhecimento facial e validação bancária operam como pedágios digitais que suspendem a cidadania dos excluídos.

Entre 26 e 40 milhões de brasileiros estão classificados em estado de hipervulnerabilidade. 78% dos idosos sofrem de analfabetismo digital profundo. 29% da população adulta é funcionalmente analfabeta. A predação burocrática intermediária extrai estimados R$ 50–100 bilhões/ano das famílias mais pobres.

## **0.2 A Resposta: Desintermediação Radical**

O ecossistema Meu Cumpadre foi desenhado como arquitetura de resposta civilizatória. Sua tese central: promover a desintermediação radical via orquestração híbrida IA + humano, objetivando uma "Inversão Cósmica" de incentivos. O Router-Ethics é o sistema nervoso autônomo que garante que essa missão não se corrompa.

## **0.3 DNA Estratégico: Jogo Infinito e Proof-First**

O ecossistema opera sob o framework do Jogo Infinito: o objetivo não é vencer competidores, mas manter-se no jogo para promover avanço contínuo em direção à Causa Justa — restituir autonomia roubada por design sistêmico. A premissa tecnológica é Proof-First, não AI-First: no universo dos hipervulneráveis, aplicar IA diretamente na camada de decisão jurídica sem validação humana robusta reproduziria a mesma tirania automatizada que o Estado impõe.

## **0.4 Modelo Access-First: Inversão de Risco**

| Nível | Nome | Preço | Objetivo |
| ----- | ----- | ----- | ----- |
| Nível 1 | Diagnóstico Cumpadre (Isca) | R$ 0 – 19,90 | Captura de dados + construção de confiança |

O Router-Ethics opera no Nível 1 como cérebro decisório que substitui a consulta de alto custo por triagem algorítmica+humana indissociável, calibrando suas redes para diagnosticar atrito, identificar exaustão emocional e injetar calor humano de acordo com a pontuação de hipervulnerabilidade.

# **Capítulo 1 — O Que É o Router-Ethics 10.0**

## **1.1 Definição e Propósito**

Router-Ethics 10.0 é o componente técnico-normativo que codifica, em regras de sistema, o que é aceitável e proibido na atuação da IA, dos Anjos e dos profissionais parceiros. Combina métricas técnicas (confidence, complexity, fraudscore), contexto social (perfil, presença de Anjo, histórico local) e regras ético-regulatórias (OAB, CFM, LGPD, CNJ) para decidir o próximo "hop" seguro de cada caso.

## **1.2 Três Blocos Hierárquicos: D → C → V**

| Bloco | Foco | Descrição | Nós |
| ----- | ----- | ----- | ----- |
| **DIGNIDADE (D)** | Vida e integridade | Choro, fome, suicídio, vergonha → IA cessa, Anjo/rede pública assume | D01–D35 |
| **COMPLIANCE (C)** | Atividade-meio | OAB/CFM/LGPD/antifraude → handoff para profissional regulado | C36–C70 |
| **VIABILIDADE (V)** | Base probatória | Zona verde C_conf ≥ 70 habilita pipeline; nó V71 exige ≥ 85 para ação final de protocolo; infra funcional → autoriza ação | V71–V105 |

*Só quando o fluxo passa ileso por D, C e V o Router permite uma ação 100% automática (ex.: protocolar benefício no Gov.br). A hierarquia é absoluta: Dignidade prevalece sobre Compliance, que prevalece sobre Viabilidade.*

## **1.3 Três Papéis Centrais**

| Papel | Função | Exemplo |
| ----- | ----- | ----- |
| Gatekeeper de Fluxo | Decide quando caso fica IA+Anjo vs. handoff obrigatório | Score > limiar → advogado |
| Firewall Ético | Impede IA/Anjo de cruzar linhas proibidas | Prometer resultado → bloqueio |
| Sensor de Anomalias | Detecta desvios: fraude, abuso, burnout | Padrão de pedágio → amputa Anjo em ms |

## **1.4 Migração v9.0 → v10.0**

| Dimensão | Router-Ethics 9.0 | Router-Ethics 10.0 |
| ----- | ----- | ----- |

# **Capítulo 2 — Arquitetura Técnica**

## **2.1 Visão Geral da Pilha**

O Router-Ethics 10.0 fica entre os Vertical AI Agents e o Model Context Protocol (MCP). Toda decisão relevante — desde interpretar um áudio rural até protocolar benefício — é submetida a uma árvore de 105 nós determinísticos que avaliam D, C e V antes de qualquer ação de alto impacto.

Fluxo técnico base: (1) Input chega (WhatsApp/API) → Intake Agent processa; (2) Vertical AI Agents preparam análise (OCR, RAG jurídico, RPA); (3) Router-Ethics avalia scores em D/C/V; (4) Se aprovado, chama MCP para falar com o mundo; se não, força handoff humano.

## **2.2 Agentes Verticais**

| Agente | Função | Integração Router |
| ----- | ----- | ----- |
| Percepção (Intake) | Transcreve áudios com sotaques NE/N, ruído, extrai entidades | Alimenta scores iniciais |
| Forense (OCR Preditivo) | Lê docs sujos, rasgados, manuscritos anos 70/80 | V75 (Legibilidade Mínima) |
| Estrategista (RAG Jurídico) | Cruza fatos com Grimório (IN 128, EC 103/2019) | Gera plano → Router avalia |
| Executor (RPA Invisível) | Gov.br: preenche, anexa, gera protocolo | Só executa após D+C+V ok |

## **2.3 Servidores MCP**

| Servidor MCP | Função | Nós de Controle |
| ----- | ----- | ----- |
| Gov.br / INSS | CNIS, status requerimentos, cartas de exigência | V71, V74, C37 |
| Financeiro (Bacen/SCR) | Auditoria consignados, descontos associativos | C41–C43, D18, D20 |
| Auditoria/Logging | Hash SHA-256 de decisões e dossiês | C51, C67, V91 |

**Nenhum agente vertical tem permissão de executar mudanças externas sem passar por pelo menos o conjunto mínimo de nós do Router (confidence, dignidade, compliance OAB/LGPD).**

*MCP = **Model Context Protocol** (JSON-RPC 2.0, especificação Anthropic). Nunca usar a expansão incorreta "Memória Persistente de Contexto" em nenhum documento MC.*

# **Capítulo 3 — Jornada E0–E7: Fluxo End-to-End do Cidadão**

A jornada do cidadão é modelada em 8 estados (E0–E7), governados pelo Router-Ethics em cada transição. O cidadão interage sempre via WhatsApp-First; o portal existe apenas como cockpit de retaguarda.

| Estado | Nome | Descrição | Router-Ethics | Critério de Saída |
| ----- | ----- | ----- | ----- | ----- |
| E0 | Sinal Fraco Captado | Contato inicial bruto, sem dados estruturados | Rota B (IA+Anjo) | 1 doc/fato + CPF + vertical |
| E1 | Triagem Básica Concluída | Scores iniciais calculados, enquadramento sugerido | A/B/C conforme scores | Scores + decisão deliberação |
| E2 | Deliberado / Não Deliberado | Plataforma assume ou não o caso | Zonas vermelha/amarela/verde | Registro incentivos ou rota alternativa |
| E3 | Dossiê Pronto | Prova coletada/validada; fixo R$84 do Anjo | V74 (completude) | Checklist probatório ok |
| E4 | Protocolo Efetuado | Requerimento protocolado (INSS/banco) | V85 + Hash SHA-256 | Nº protocolo capturado |
| E5 | Acompanhamento | Monitoramento de prazos, exigências, perícias | V86 + D33 (fadiga Anjo) | Exigências traduzidas em tarefas |
| E6 | Desfecho Comunicado | Resultado metabolizado com o usuário | D29/D30 (proteção/celebração) | Compreensão verificada |
| E7 | Pós-Desfecho / Soberania | Lições, autonomia, fechamento | D25 (Missão Cumprida) | Pessoa explica jornada sozinha |

## **3.1 Integração com a Correção Narrativa "48 Horas"**

**IMPORTANTE: "48 horas" é o sprint interno do Meu Cumpadre entre E0 (início da triagem) e E4 (protocolo no INSS), quando toda documentação está completa. NÃO significa que o benefício sai em 48h. O tempo médio de análise do INSS (BEPS Jan/2026) é de 57 dias após o protocolo. A ressalva fundamental: geralmente é necessário que a persona providencie documentações de outras verticais (Cartório, Saúde, Rural) — esse tempo é variável.**

## **3.2 Rotas A/B/C por Estado**

| Rota | Configuração | Quando | Exemplo |
| ----- | ----- | ----- | ----- |
| **A** | IA sozinha | Baixa complexidade + boa literacia digital | Extrato CNIS simples |
| **B** | IA + Anjo (padrão) | Hipervulneráveis em geral, complexidade moderada | Aposentadoria rural por idade |
| **C** | IA + Anjo + Profissional regulado | Alta complexidade jurídica/médica, fraude, litígio | Revisão da vida toda, BPC com perícia |

## **3.3 Caso Não Deliberado: Dever de Cuidado**

Quando o Router barra o caso (zona vermelha), o sistema NÃO abandona a pessoa. O Anjo explica em linguagem digna, oferece pelo menos uma rota alternativa real (CRAS, Defensoria, sindicato, pastoral), e planta "microsoberania" (como guardar documentos para tentativa futura). Regra escrita: qualquer fluxo que aumente dependência ou culpa do usuário é suspeito; o design correto aumenta compreensão, escolha e controle.

# **Capítulo 4 — Motor de Decisão: Scores, Zonas e Thresholds**

## **4.1 Os Três Scores (0–100)**

| Score | O Que Mede | Inputs Principais | Threshold Crítico |
| ----- | ----- | ----- | ----- |
| Confidence (C_conf) | Chance técnica de deferimento | CNIS + docs + enquadramento legal | ≥ 70 verde; 40–70 amarela; < 40 vermelha |
| Complexity (C_comp) | Dificuldade técnico-jurídica | Nº normas, exceções, órgãos, risco OAB/CFM | Alto → Rota C obrigatória |
| Fraudscore (C_fraud) | Probabilidade de fraude | CPMI, consignados, associações, docs suspeitos | Alto → bloqueio pipeline + investigação |

**HANDOFF obrigatório se QUALQUER dos três scores cruza seu threshold. AUTONOMIA somente se todos passam simultaneamente.**

## **4.2 Zonas de Confidence e Impacto**

| Zona | Faixa | Pipeline Pago | Incentivo Anjo | Ação |
| ----- | ----- | ----- | ----- | ----- |
| **VERMELHA** | < 40 | Bloqueado | Nenhum | Orientação + CRAS/Defensoria |
| **AMARELA** | 40–70 | Sob análise | Após reclassificação | Triagem aprofundada pelo Anjo |
| **VERDE** | ≥ 70 | Liberado | R$84 fixo + micro-share 5% | Caso Meu Cumpadre → E3+ |

## **4.3 Modelo de Incentivos do Anjo (Atrelado aos Scores)**

O modelo anti-avalanche garante que Anjos ganham mais quando a triagem é boa e ética, não quando empilham volume:

| Incentivo | Condição | Valor | Teto |
| ----- | ----- | ----- | ----- |
| Bônus fixo | Zona verde + deliberado + E3 (Dossiê Pronto) | R$ 84,00/caso | — |
| Micro-share | Zona verde + bem-sucedido | 5% do success fee | R$ 1.200/mês; R$ 3.600/ano |
| Caso vermelho | Sempre | R$ 0 | Sem incentivo |
| Caso amarelo | Só se reclassificado p/ verde | Conforme verde | Conforme verde |

> **⚠ NOTA EDITORIAL v3.0:** O modelo de incentivos descrito nesta seção (bônus fixo R$84 + micro-share 5% de success fee) está em revisão sob ADR-007 v3.0 (20/04/2026), que vedou o success fee percentual. A selagem final ocorrerá em v3.1 após parecer da Dra. Juliana Pereira de Melo (19/05/2026). O **mecanismo decisório do Router** (scores, zonas, thresholds) **permanece válido independentemente do modelo de remuneração do Anjo** — a arquitetura ética é agnóstica ao modelo comercial.

## **4.4 Transparência Algorítmica: "Não é Culpa Sua"**

Os scores são traduzidos em linguagem de quintal, nunca em números. Em vez de "seu confidence é 38", o Anjo/IA diz: "Com os papéis que o senhor tem hoje, a regra do INSS quase não deixa brecha. Se eu prometesse agora, eu estaria mentindo pro senhor." Vermelho = "muito arriscado com o que temos hoje". Amarelo = "podemos tentar completar melhor as provas". Verde = "temos base boa, mas não é 100% garantido."

# **Capítulo 5 — Compliance: OAB, LGPD, CFM**

## **5.1 Firewall Atividade-Meio vs. Atividade-Fim**

O Router-Ethics é a muralha que impede enquadramento por exercício ilegal da advocacia ou violação de sigilo. O Meu Cumpadre atua apenas em atividade-meio (organizar provas, traduzir informações, protocolar dados). Quando a única saída é litigar ou construir tese interpretativa, o Router encerra sua jurisdição e aciona handoff.

## **5.2 Matriz de Compliance OAB × Router-Ethics**

| Ação/Fala | IA | Anjo | Advogado OAB | Nós Router |

## **5.3 LGPD: Nós Específicos**

C48 (Consentimento Granular): registro específico com hash SHA-256 em MCP. C49 (Minimização): só coleta o estritamente necessário. C50 (Revogação): usuário pode retirar consentimento. C51 (Registro Criptográfico): todo dossiê final traz hash no rodapé. C61 (Proteção contra Vazamento): resposta a incidentes com revogação automática.

# **Capítulo 6 — Casos Práticos e Simulações**

## **6.1 Zilda Rural — Aposentadoria por idade (Rota B → Verde)**

**Contexto:** Zilda, 61 anos, NE rural, fichas de sindicato manchadas de umidade, nunca contribuiu formalmente. Chega via Anjo comunitário com áudio em sotaque sertanejo pesado.

E0/E1: Áudio transcrito + OCR nas fichas. Nós acionados: D05 (Analfabetismo Digital), D21 (Tempo Humano), V75 (Legibilidade). Scores: confidence 72 (verde), complexity 55 (moderada), fraud 10 (baixa).

E2: Deliberado Rota B (IA+Anjo). OCR Preditivo encontra inconsistência: certidão de 1985 diz "comerciante" em vez de "lavrador". V78 (CNIS×Provas) bloqueia na origem. Anjo recebe alerta: "A prova material falhou. Precisamos da declaração do proprietário da terra."

E3–E4: Dossiê completo após nova prova. Protocolo INSS. E5: Monitoramento 57+ dias. E6/E7: Deferimento. Anjo celebra: "Zilda, a senhora lutou por isso. Não foi presente do Cumpadre, foi conquista sua."

*Resultado: Torre Reta probatória completa antes do protocolo. Inversão estatística: não protocolamos para "ver se passa"; protocolamos porque a estrutura lógica não deixa margem para negação.*

## **6.2 Seu João — Fraude consignado (Rota C → Handoff → Verde)**

**Contexto:** João, 72 anos, NE rural, aposentadoria R$ 1.412, desconto associativo R$ 150 que desconhece. "Não sei o que é isso, tô comendo só arroz."

Nós D02 (Fome Crônica), D06 (Sobrecarga Emocional), C42 (Antifraude Consignado). Scores: confidence 80 (verde), complexity 75 (alta financeira), fraud 60 (média). Router detecta risco OAB/Bacen → Rota C. Handoff cirúrgico: JSON + resumo → advogado parceiro. Descontos bloqueados. R$ 150/mês recuperados. João aprende a verificar extrato.

## **6.3 Dona Francisca — Perícia inacessível (Caso Não Deliberado → Vermelha)**

**Contexto:** Francisca, 55 anos, PcD física/intelectual, BPC travado, mora 50km do posto INSS sem transporte. "Não ando, vizinho diz que vou perder tudo."

Nós D17 (PcD), D24 (Confusão Persistente), D03 (ideação implícita). Scores: confidence 35 (vermelha), complexity 90, fraud 5. Router barra pipeline pago. Dever de cuidado: rota alternativa (perícia domiciliar via Defensoria + médico SUS), microsoberania ("Guarda esta carta e anota a data"). Francisca sai com plano concreto, sem culpa ou abandono.

## **9.2 Circuit Breaker Fênix: Triggers Imediatos**

Independente dos três scores calculados, os seguintes triggers forçam **HANDOFF imediato** (automação abortada, log permanente no Tier 4 do MCP, handoff urgente ao Anjo/Advogado):

1. Risco de vida, violência, fome (D01, D02, D03)
2. Idade > 80 + analfabetismo digital + isolamento (D04 + D05 + D18)
3. 2+ indeferimentos no mesmo benefício (histórico detectado via CNIS)
4. Fraude ativa detectada contra o usuário (C41, C42)
5. Solicitação explícita de atendimento humano pelo cidadão
6. Áudio com choro, desespero ou confusão mental (D01, D06, D24)

A frequência esperada de ativação do Fênix é baixa (< 0,5% das interações). Sua existência é inegociável: é a garantia de que, mesmo que todos os outros mecanismos falhem, existe uma última linha de defesa que prioriza a pessoa sobre o processo.

## **9.3 Horizonte v11.0 (Visão)**

Com base nos aprendizados do BEPS Jan/2026 e das simulações de campo, a v11.0 poderá incorporar: (1) Fine-tuning do Llama 4 Scout com dados do Grimório para NLP sertanejo nativo; (2) Nós específicos para a vertical Consumidor (consignados) dado o volume de fraude CPMI; (3) Integração com WhatsApp Cloud API oficial (migração de Evolution API); (4) Calibração de thresholds com dados reais dos primeiros 500 casos piloto (pós-Fase Crisálida).
