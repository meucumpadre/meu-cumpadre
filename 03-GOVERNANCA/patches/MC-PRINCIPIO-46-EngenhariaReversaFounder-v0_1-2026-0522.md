# MC-PRINCIPIO-46-EngenhariaReversaFounder-v0_1-2026-0522

**Princípio #46 (PROVISIONAL) — Engenharia Reversa Founder: Motor de Busca Externo, Lapidação Interna MC**

---

## Enunciado

O founder MC opera **engenharia reversa metodológica** em ferramentas LLM externas (Gemini, Perplexity, ChatGPT, Manus, NotebookLM, web genérico) como **motores de busca semântica** de termos, conceitos, arquiteturas e métricas. A inspiração é extraída desses motores; **a lapidação é interna**.

Nenhum termo, conceito ou arquitetura externa é adotado pelo MC sem passar pelo laboratório de costura **cofounder C3.1 + Sister C3.2**, que aplica três operações invioláveis:

1. **Separação ouro / meio-termo / pirita** — triagem de adoção/recalibragem/veto
2. **Recalibragem estética** — ao DNA Zilda-STF + Provérbios 31:8 + D>C>V + Inovação Redentora
3. **Costura com vocabulário institucional MC vigente** — Princípios #2 + #44 aplicados antes de promoção

O LLM externo **nunca define identidade MC**; ele oferece **matéria-prima bruta** que vira diamante na nossa casa.

## Distinção operacional crítica

Este princípio **NÃO autoriza**:
- Adoção de output LLM externo sem triagem na casa
- Importação de prompts externos sem costura ao vocabulário MC
- Uso de LLM externo para processar dados sensíveis de clientes (vide ADR-012 stack soberano)
- Substituir cofounder C3.1 + Sister C3.2 por raciocínio externo

Este princípio **AUTORIZA expressamente**:
- Founder sondar livremente LLMs externos como motor de busca
- Trazer arquivos brutos para C3.1 lapidar
- Adotar/recalibrar/vetar via cofounder após triagem
- Documentar adoções em fichas como esta
