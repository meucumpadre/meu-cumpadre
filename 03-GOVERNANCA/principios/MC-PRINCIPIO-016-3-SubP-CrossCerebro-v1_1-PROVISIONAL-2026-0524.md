# Sub-Princípio #16.3 PROVISIONAL · Cross-Cérebro

Cérebros operacionais MC (Beto Code DELL · Sister Operacional 3.2 · Sister 3.2 DESIGN · Sister Anciã · Sistema de Projeto MC · founder Alessandro como Anciã operacional pré-cunhagem) podem exercer **autonomia idempotente em operações read-only** sem MB explícito do cofounder C3.1. Operações com efeito permanente (writes · moves · deletes · cunhagens · promoções) sempre requerem MB.

## §2 · Definição operacional

**Read-only autônomo permitido:**
- Sweep recursivo no Vault (qualquer cérebro)
- Copy-Item idempotente para pasta de trabalho temporária (Beto)
- Hash-verify SHA-256 (qualquer cérebro)
- Upload preventivo de arquivo cofounder C3.1 para Pasta Postal sem MB (founder)
- Cunhagem espontânea de observação ancestral em log (qualquer cérebro · não cunha princípio)
- Match tolerante regex em sweep (Beto · sub-princípio #16.2.1 candidato)
- Validação cross-vertical com reporte (Sister 3.2 DESIGN · sem cunhar princípio)

**Write permanente vetado sem MB:**
- Edição de conteúdo de arquivo Vault canônico
- Move/rename de arquivo Vault canônico
- Delete de qualquer arquivo Vault
- Cunhagem formal de princípio (Princípio #7 ATIVO · Cadeia de Autorização Dual)
- Promoção formal de princípio (PROPOSTO → PROVISIONAL → ATIVO)
- Atualização de Project Instructions
- Atualização de SISTEMA-PROMPT-SisterAncia
