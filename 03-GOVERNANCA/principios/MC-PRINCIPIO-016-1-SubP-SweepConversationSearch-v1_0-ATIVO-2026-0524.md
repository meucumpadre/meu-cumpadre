# Sub-Princípio MC #16.1 ATIVO

## Enunciado

Antes de propor qualquer cunhagem de Princípio #N (ATIVO, PROVISIONAL ou PROPOSTO) na sessão Claude.ai cofounder, é obrigatório executar `conversation_search` com queries calibradas para:
- (a) o número do slot proposto
- (b) o nome conceitual do princípio
- (c) keywords semânticas adjacentes

Resultado negativo libera cunhagem. Resultado positivo dispara protocolo do Princípio #26 (anti-colisão pré-cunhagem · 7 passos).

Esta verificação é responsabilidade exclusiva do C3.1 antes de despachar qualquer MB que inclua cunhagem de princípio para o Code DELL.

## Sanção por violação

Retrabalho cofounder + errata documentada + custo carregado pelo C3.1 (não terceirizado).

## Histórico de violações documentadas

- **18/05/2026:** cunhagem #44 MarcaFilledLinha sem sweep · detectado 24/05 · renumeração para #46 · errata `MC-ERRATA-Principio-044-Colisao-001`

## Relação com Princípio #26

Princípio #26 é o **protocolo de 7 passos** (institucional). Sub-princípio #16.1 é a **responsabilização nominal** do cofounder C3.1 pelo cumprimento do protocolo. Os dois são complementares · não substituíveis.
